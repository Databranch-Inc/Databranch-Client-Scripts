1. Brand Origin & Design Philosophy

Source Material

This design system derives from three authoritative brand sources: the Databranch logo, the Databranch website at databranch.com, and the existing internal tooling palette established in Start-ScriptManagementBrowser.ps1 and the Start-EventLogCollection documentation.

The logo establishes the two core brand colors: a bold crimson red used for the wordmark and accent spheres, set against a deep circuit-board blue rendered as glowing electric-blue traces. Black is used as the deepest background. The overall character is technical, high-contrast, and dark-themed ‚Äî the design system extends these qualities directly into application and documentation UI.

Philosophy: The Circuit Board Principle

Databranch's visual identity evokes circuit board imagery ‚Äî deep dark backgrounds, precise glowing traces of blue light, and vivid focal points of red. The UI design system translates this directly:

  • Dark backgrounds first. All surfaces are deep navy-black. There is no light mode variant. The depth stack creates a sense of layered physical structure, like looking at hardware under illumination.
  • Blue as the primary interactive signal. Electric blue is the color of active elements, links, focus rings, and interactive affordances ‚Äî exactly as it is in the logo's circuit traces.
  • Red as a power accent, used precisely. The brand red never competes for attention ‚Äî it marks the most important single element in a given context: the primary action, the brand identity label, the critical alert. It is not a general-purpose highlight color.
  • Desaturated red for comfort. The logo's vivid #C8102E red is aggressive at small sizes on dark backgrounds. The design system shifts to #C0404A (soft red) and #D07080 (pale red) for most UI applications, reserving the vivid red only for brand-identity contexts.
  • White text with a blue undertone. Pure white (#FFFFFF) reads as harsh and disconnected from the blue-dark palette. All text whites use #F0F4FF ‚Äî a near-white with a subtle blue cast that integrates naturally with the surface palette.
üé®
Design Decision Record
When re-applying this spec to any document or application, the central question to ask first is: what is the primary interactive signal on this surface? That element gets accent blue. What is the single most important brand or identity marker? That gets red. Everything else is layered in the neutral blue-dark surface stack. Do not distribute red broadly ‚Äî one red element per visual zone is the rule.

Existing Palette Analysis

The prior WPF application used a Dracula-inspired purple-dark palette (#1E1E2E, #252537, #7B9CFF). The existing HTML documentation used a navy-blue palette that was already closer to the brand. This design system converges both onto a unified navy-to-void depth stack with brand red and electric blue accents, retiring the purple undertones in favor of pure blue-dark tones.

Field labels sit above the input. They use uppercase, 10px, font-weight: 600, Text Muted color (#607090), with 4px of margin below. This uppercase small-caps treatment creates visual hierarchy without competing with the input content.

Panel vs Card Distinction

Design Decision Record
The active nav link indicator is red in the Databranch design system, not blue. The prior HTML documentation used a blue active indicator. This was changed to use Brand Red Soft as the active sidebar accent ‚Äî it anchors the sidebar to brand identity and clearly differentiates "where you are" from "what you can click" (interactive blue). Scroll-Spy Navigation (HTML)

Navigation links highlight automatically as the user scrolls using IntersectionObserver with rootMargin: '-20% 0px -70% 0px'. Active class is applied to the nearest matching href="#id" link. This rootMargin targets the upper third of the viewport, ensuring the active link reflects what the reader is currently engaging with.

WinForms Dark Mode Limitation
WinForms controls do not natively support dark mode on Windows. Many controls (ComboBox, ScrollBar, CheckBox, MenuStrip) require custom owner-draw implementations or third-party dark-mode libraries (e.g. DarkUI, MaterialSkin) to achieve consistent dark theming. Plan for additional implementation effort when using WinForms for complex UIs. WPF is strongly preferred for new Databranch desktop applications due to its full styling support.

Avoiding Pipeline Pollution

Functions that both write log messages and return values must route Write-Log calls to | Out-Null and use unary comma operator (return ,$list) to prevent log strings from being captured as return values. This is a critical PS 5.1 gotcha ‚Äî see script comments for the full pattern.

21. Applying This Spec to Existing Work

When Redesigning the WPF App (Start-ScriptManagementBrowser)

  • Replace the current purple-dark surface stack (#1E1E2E, #252537, #2E2E45, #3A3A55) with the new navy-dark surface stack (Void ‚Üí Base ‚Üí Raised ‚Üí Card ‚Üí Elevated).
  • Replace the current accent blue (#7B9CFF ‚Äî a periwinkle/lavender blue) with Brand Blue Bright (#2E8BFF) and update the AccentButton to use AccentRed (#C0404A) as the primary action button.
  • The existing AccentGreen (#50FA7B) SyncButton was a good UX choice for the Sync action ‚Äî retain green for that specific status-success affordance but shift to #22C55E to match the unified semantic system.
  • Replace TextMuted (#6272A4 ‚Äî bluish-purple) with the new #607090 (clean blue-gray muted).
  • Replace BorderColor (#44446A ‚Äî purple-tinted) with BorderDefault (#213A58).
  • Replace PS1Color (#BD93F9 ‚Äî purple) file type badge with AccentBluePale (#4A8FD4) ‚Äî purple is now reserved for DEBUG only.

When Redesigning HTML Documentation

  • The existing HTML documentation navy palette is already very close. Primary changes: shift sidebar active indicator from blue to Brand Red Soft. Shift H3 from teal to Brand Red Pale. Shift code block left-border from blue to red. Update --navy, --navy-mid, --navy-light to the new surface token values.
  • The existing teal (#00C9B1) used for company labels, H3, and arch tags should be retired. The design system does not include teal ‚Äî it is not a Databranch brand color. Replace with Brand Red Pale for labels and Brand Blue Pale for tags.
  • Cover eyebrow changes from teal to Brand Red Soft. Cover gradient deepens slightly toward void-black.
  • All other component shapes, spacing, and structure remain identical.

Priority Order for Applying Changes

When updating existing documents or applications, apply changes in this order to minimize visual disruption and ensure correctness at each step:

  • 1. Surfaces first. Update all background colors to the new surface stack. This creates the correct depth foundation everything else sits on.
  • 2. Text colors. Update all text to the new text token hierarchy. This restores readability immediately.
  • 3. Borders. Update borders to the new border tokens. This cleans up the structural definition of panels and cards.
  • 4. Accent colors. Apply Brand Red Soft to primary actions and active indicators. Apply Brand Blue Bright to interactive and H2 elements.
  • 5. Status/semantic colors. Update severity indicators, callouts, and status messages to the unified semantic palette.
  • 6. Code blocks. Update code block backgrounds, syntax highlight colors, and left-border accent to red.
